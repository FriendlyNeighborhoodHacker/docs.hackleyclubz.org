# Key Admin Flows

These flows cover the work done by faculty advisors and administrators.

## 1. Approving a new club

1. A student or faculty member submits a **club proposal**.
2. An administrator reviews the proposal details.
3. The admin **approves** or **declines** the proposal.
4. When approved, the club becomes visible in the directory.

## 2. Approving members

1. A student requests to join a club.
2. The club's advisor sees the pending request.
3. The advisor **approves** or **declines** the request.
4. The student is notified of the decision.

## 3. Moderating communication

1. Advisors monitor announcements and chat for their clubs.
2. They can remove inappropriate content.
3. They can mute or remove a member if needed.

## 4. Managing users

1. An administrator opens the user management area.
2. They can assign roles (e.g. promote a student to club leader).
3. They can deactivate accounts (e.g. for graduated students).

## 5. Overseeing the program

1. Administrators view all clubs and their status.
2. They track which clubs are active vs. inactive.
3. They use reporting to monitor participation (see **Reporting Flows**).
